## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**File**: `extensions-node/about/src/package_json.rs:136-157`

**What**: normalize_repo_url does not handle `git+ssh://git@host/owner/repo.git`: it strips the git+ prefix and returns `ssh://git@host/owner/repo`, leaving an unbrowsable scheme and embedded user-info in the repository identity.

**Why it matters**: package.json repository.url commonly carries SSH form when generated by `npm init` against an SSH remote. The output is neither a valid web URL nor consistent with the GitHub remote fallback (which converts to https://). Downstream UI shows ssh://git@... links.
<!-- SECTION:DESCRIPTION:END -->
